ℹCreating an event is the first step in the full NKB Events workflow. Everything else — registration forms, certificates, event passes — is tied to the event you create here.
01
Navigate to Event Creation REQUIRED
  • Log in to the Django Admin Panel with your admin credentials
  • In the left sidebar or main menu, navigate to NKB Events → Events
  • You will see a list of all existing events with their status
  • Click "Add Event" in the top-right corner to open a blank event form
ℹIf you don't see NKB Events in the menu, your admin account may not have the required permissions. Contact the portal administrator to get access.
02
Fill in Basic Event Details REQUIRED
FieldWhat It DoesRequired
Event NameThe full display name students see in the app. Keep it clear and descriptive, e.g. "NxtWave AI Workshop — June 2026"Yes
Short TextA one-liner shown on the event card in the student portal. Summarize the event's key value in under 80 characters.Yes
DescriptionFull event description visible on the event detail page. Can include agenda, speaker info, prizes, or any relevant context for students.Yes

03
Configure Visibility & Eligibility
FieldWhat It DoesValues
Visibility StatusControls whether the event appears in the student portal. Keep it DRAFT while setting up, change to PUBLIC when ready to launch.DRAFT / PUBLIC
Participation TypeDetermines if students join individually or as a team. INDIVIDUAL events register one student per entry. TEAM events group students under a team leader.INDIVIDUAL / TEAM
Enroll Plans StrRestricts who can see and register for the event. Use NIAT to limit to NIAT students, or leave blank / use the appropriate plan code for other cohorts.NIAT etc.

📌Always set Visibility to DRAFT while you're still configuring the event. Change to PUBLIC only after the registration form, submission form, and certificates are all set up.
04
Save and Copy the Event ID CRITICAL
  • Click Save at the bottom of the form
  • After saving, the page reloads to the event detail view
  • Look at your browser's address bar — the URL will contain the Event ID as a number, e.g. /admin/nkb_events/event/142/change/
  • Copy this number and paste it somewhere safe (Notepad, Slack message to yourself)
  • You will need this ID later when setting up Certificate Template Data in Module 05
⚠If you forget to copy the Event ID, you can always come back to the event list, open the event, and read it from the URL again. But copying it right now saves time later.
05
Common Mistakes to Avoid
  • Wrong Visibility: Setting to PUBLIC too early means students see an incomplete event. Always use DRAFT while setting up.
  • Wrong Enroll Plans Str: If this field is mistyped, eligible students won't see the event at all. Double-check the plan code.
  • Start date in the past: The system may show the event as already ended. Always verify the start/end dates match the actual event schedule.
  • Forgetting the Event ID: You'll need it for certificate setup. Copy it immediately after saving.
✓Once saved with the correct settings, the event is ready for Module 02 (Registration Form) and Module 03 (Submission Form) configuration.

ℹThe Registration Form is the information students fill in when they click "Register" for your event. It helps you collect details like college name, year of study, or contact info before the event starts.
01
Open the Event for Editing
  • Go to NKB Events → Events in the Django Admin Panel
  • Find the event you created in Module 01 — use the search bar or scroll to locate it
  • Click the event name or the Edit pencil icon to open the edit form
  • Scroll down past the basic event fields to find the Event Registration Form section — it appears below the main event fields
02
Understanding When to Add a Registration Form
  • Add a form when you need to collect specific student data before the event (e.g. college, branch, year of study, phone number, project idea)
  • Skip the form for simple awareness events or internal meetups where registration without data collection is sufficient
  • The form data you collect becomes accessible in the Registrations list — useful for filtering and organizing participants
📌If no registration form is added, students can still register for the event but won't be prompted to fill in any details. This is fine for simple events but not for competitions or hackathons where you need participant info.
03
Add the Registration Form KEY STEP
  • In the Event Registration Form section, click "Add another Event Registration Form"
  • A dropdown or inline form will appear — select the existing form that matches your event type, or work with the team to create a new form in the form builder
  • The form you attach will be shown to students when they click "Register" on the event
  • Typical form fields used for NIAT events: Full Name, College Name, Branch, Year of Study, Mobile Number, NW User ID
ℹYou can only attach forms that already exist in the system. If you need a new custom form built for a specific event, coordinate with the tech team to get it created first.
04
Save and Verify
  • After selecting the form, click Save at the bottom of the event page
  • To verify: Open the event as a student would (in the student portal or a test account) and confirm the registration form appears when clicking "Register"
  • Check that all the expected fields are present in the form
✓Once the registration form is attached and saved, students who register will see these fields. Their responses are stored and visible in the Registrations section — covered in Module 04.

ℹThe Submission Form is different from the Registration Form. Registration Form = data collected when a student signs up. Submission Form = what students upload or fill in to submit their actual project or deliverable for evaluation.
01
Know When to Use a Submission Form
  • Use this for hackathons, competitions, or project-based events where students need to submit a deliverable
  • Typical submission fields: GitHub repo link, deployed project URL, video demo link, project description, team member details
  • Do NOT add a submission form for simple attendance-based events like webinars or workshops — it is unnecessary
  • The submission form appears to students within the event, typically as a "Submit Project" button visible after they register
02
Locate the Submission Form Section
  • Open the event in edit mode from NKB Events → Events
  • Scroll down below the Registration Form section — you will find the Event Project Submission Form section separately
  • Both forms live on the same event edit page but are distinct sections
📌The Registration Form and Submission Form serve completely different purposes. A student fills the Registration Form when they sign up. They fill the Submission Form when they submit their completed project — usually at the end of the event.
03
Add the Submission Form KEY STEP
  • Click "Add another Event Project Submission Form" within the section
  • Select the appropriate submission form from the dropdown — this form must already be created in the form builder
  • The form you attach will be the one students use to submit their project deliverable
  • Coordinate with the tech team if you need a new submission form created specifically for this event
04
Configure Deadline and Save
  • If the submission form section allows configuring a submission deadline, set the exact date and time after which submissions will be closed
  • Submissions after the deadline will not be accepted, so make sure the deadline is correct before going live
  • Click Save at the bottom of the event page to apply all changes
  • Verify by switching to the student view — you should see the "Submit Project" button appear on the event page after registering
✓Once saved, registered students will be able to see and fill the submission form from their event page. All submissions are stored and accessible to admins for evaluation.

Approve & Reject Registrations
Review student registrations and manage their status in the Admin Panel
▶Module 04 — Approve & Reject WalkthroughWatch before the quiz
▶

Video will be embedded here

ℹWhen students register for an event, their registrations appear in the Admin Panel with a Pending status by default. You review each registration and either Approve or Reject it. Approval gives the student access to the event; rejection blocks access.
01
Access the Registrations List
  • Go to NKB Events → Events and find your event
  • Open the event, then scroll to the Registrations section — or look for a View Registrations link
  • Alternatively, go to NKB Events → Registrations and filter by your event name
  • You will see a list of all registrations with columns: Student Name, NW ID, Registration Date, Status, and Form Responses
02
Review a Registration Entry
  • Click on a student's registration entry to open its detail view
  • Review the information the student filled in the registration form — check for completeness and eligibility
  • Verify details like: Are they a valid NIAT student? Did they fill all required fields? Is their contact information correct?
  • The current Status will show as PENDING until you take action
📌Some events may have specific eligibility criteria — e.g. only 2nd year students, or only students from a specific batch. Verify against the event rules before approving.
03
Approve a Registration APPROVE
  • In the registration detail view, locate the Status dropdown field
  • Change it from PENDING to APPROVED
  • Click Save at the bottom of the page
  • The student's event access is now activated — they can see the event as live in their portal
  • If Event Pass is configured (Module 06), the student will also receive their personalized pass automatically upon approval
✓Approved students can log in and see the event as active. If an Event Pass was configured, they'll receive it immediately. This is the most common action you'll take.
04
Bulk Approve Multiple Registrations
  • On the Registrations list view, check the checkboxes next to multiple registrations
  • Use the Action dropdown at the top to select "Approve selected registrations"
  • Click Go — all selected registrations are approved in one operation
  • This is useful when a large batch of students register at once and all are eligible
ℹBulk approve saves significant time when handling 50+ registrations. Always do a spot-check of a few entries before bulk approving to ensure eligibility.
05
Reject a Registration REJECT
  • Open the registration entry and change the Status from PENDING to REJECTED
  • Fill in the Notes field with the reason for rejection — this is visible to admins and helps future reference (e.g. "Not an NIAT student", "Incomplete form", "Duplicate registration")
  • Click Save
  • The student will no longer have active access to the event
✕Rejection is not permanent. You can reverse it at any time by opening the same registration and setting the status back to APPROVED. Always add a note explaining the reason — it helps when someone follows up.
06
Re-Approve a Rejected Student
  • If you need to reinstate a student whose registration was rejected, open their registration entry
  • Change the Status from REJECTED back to APPROVED
  • Click Save — the student regains full event access immediately
  • Update or clear the Notes field to reflect the reinstatement
✓No new registration needed. Simply editing the status to Approved is sufficient to restore access. The student's original form data is preserved.

📌
Prerequisite: The certificate template must be configured in the Poster Generation service first. A Certificate Name Enum will then be available — keep it handy before starting this module.
ℹCertificate generation is a 2-part process: (1) Enable it on the event itself and set the attendance threshold, then (2) Create a Certificate Template Data entry that links the event to the correct certificate design. Both parts must be correct for certificates to generate automatically.
01
Enable Certificate on the Event MASTER SWITCH
  • Open your event in edit mode: NKB Events → Events → [your event]
  • Scroll down to find the Is Certificate Enabled checkbox
  • Check this box to activate certificate generation — this is the global on/off switch
  • If this is not checked, no certificate will be generated for anyone, regardless of any other settings
  • Click Save after enabling it
📌Enable this checkbox before the event starts. Enabling it after students have completed the event will not retroactively issue certificates to those who already attended.
02
Set Resource ID and Minimum Threshold CRITERIA
FieldWhat It DoesGuidance
Resource IDLinks the event to an LMS content unit — the system uses this to track how much of the content a student completed.Leave blank for standalone virtual meetups that have no LMS resource.
Minimum Threshold (%)The percentage of event content a student must complete to earn their certificate. Example: 50% on a 2-hour event = 1 hour attended minimum.If left blank, defaults to 1% — meaning nearly everyone who opens the event gets a certificate. Set a meaningful value (e.g. 50–80%) for workshops.
⚠Students who don't meet the Minimum Threshold will NOT receive a certificate. Set this carefully — too high excludes deserving students; too low makes it meaningless.
03
Create a Certificate Template Data Entry SECOND PART
  • In the Django Admin Panel, navigate to Certificate Template Data (under NKB Events or Certificates section)
  • Click Add Certificate Template Data to create a new entry
  • Fill in the following three fields:
FieldWhat to Enter
Certificate Resource IDPaste the Event ID you saved in Module 01 (from the event URL). This links the certificate config to your specific event.
Template DataA JSON object identifying the certificate design. Use the template below exactly.
Enrollment PlansEnter NIAT for NIAT events. This limits certificate generation to only students enrolled in this plan.
 
{
  "certificate_template_type": "DEFAULT",
  "certificate_name_enum": "NIAT_MURFAI_CERTIFICATE"
}
⚠The certificate_name_enum value must exactly match the enum set up in Poster Generation — character for character, including underscores and uppercase. Any mismatch means zero certificates are generated. Always copy-paste this value, never type it manually.
04
Certificate Delivery and Troubleshooting
  • Certificates are not instant — they are generated in a batch process that runs after the event ends
  • Eligible students receive their certificate by end of day on the event date
  • Students can view and download it from the Achievements / Certificates section in their NxtWave app
ℹ
If students say they didn't receive a certificate, check in order:
1. Is Is Certificate Enabled checked on the event?
2. Did the student meet the Minimum Threshold?
3. Is the certificate_name_enum in Template Data correct and matching Poster Generation?
4. Is the Certificate Resource ID set to the correct Event ID?

📌
Prerequisite: The pass template must be configured in the Poster Generation service first. A Poster Name Enum will then be available — get this value before proceeding.
ℹAn Event Pass is a personalized digital card (like a conference badge) that each registered student receives automatically when their registration is approved. It typically shows their name, photo, and event details overlaid on a branded poster design created by the design team.
01
Open the Event Pass Config Section
  • Open your event in edit mode: NKB Events → Events → [your event]
  • Scroll down to find the Event Pass Config section — it appears below the certificate settings
  • This section is where all pass-related configuration lives
  • If you don't see the Event Pass Config section, check that the event is saved and your account has permission to configure passes
02
Enter the Poster Name Enum CRITICAL
  • In the Event Pass Config section, find the Poster Name Enum field
  • Enter the exact enum value that was set up in the Poster Generation service for this event's pass template
  • This value is usually in ALL_CAPS_WITH_UNDERSCORES format, e.g. NIAT_JUNE_EVENT_PASS
  • Get this value from the design/tech team who created the template — do not guess it
  • Any typo, extra space, or case mismatch will cause the pass to silently fail — students will not receive anything
⚠Always copy-paste the Poster Name Enum from the source. Never type it manually. Even one wrong character prevents the pass from generating for all students.
03
Select the Overlay Options MUST MATCH TEMPLATE

Overlay options tell the system which dynamic data to print on top of the poster background. Only enable the overlays that the pass template was designed to support — check with the person who created the template.

OptionWhat It Overlays on the PassNotes
Overlay NameThe student's full name, personalized on each passEnable for almost all event passes — makes the pass personal
Overlay Profile PicThe student's profile photo from their NxtWave accountEnable if the template has a photo slot. Requires Fallback Avatar URL (see Step 4)
Overlay Event Start DateThe event's start date printed on the passEnable if the template includes an event date display area
Overlay Event End DateThe event's end date printed on the passEnable for multi-day events where the template shows date range
Overlay Issued DateThe date the pass was generated and issuedLess common — only if the template has an "Issued on" field
📌Only select overlays that the poster template was designed for. If you enable "Overlay Profile Pic" but the template has no photo slot, the pass layout will break or text will be printed in the wrong place.
04
Set the Fallback Avatar URL CONDITIONAL
  • This field is only relevant when Overlay Profile Pic is enabled
  • Some students may not have uploaded a profile photo to their NxtWave account
  • In that case, the system uses the Fallback Avatar URL as a placeholder image on their pass
  • Provide a direct link to a generic avatar or brand logo image (must be a publicly accessible URL ending in .png or .jpg)
  • If this field is left blank and a student has no profile pic, their pass may render with a broken image or empty space
ℹUse a neutral placeholder image like a silhouette avatar or the NxtWave logo as the fallback. Avoid using a specific person's photo as the default.
05
Save and Verify the Pass Setup
  • After filling in Poster Name Enum, selecting overlays, and setting the fallback URL, click Save
  • To test: approve a test registration (your own or a colleague's) and check if the event pass appears in the student's event view
  • The pass should show the student's name (and photo if enabled) on the branded template background
  • If the pass does not appear within a few minutes of approval, verify the Poster Name Enum matches exactly what's in Poster Generation
✓Once configured correctly, passes are fully automatic. Every new approved registration instantly receives a personalized pass — no manual action needed per student. The pass remains accessible in the student's event page.
