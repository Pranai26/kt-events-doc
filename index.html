▶Module 04 — Approve & Reject WalkthroughWatch before the quiz
▶

Video will be embedded here

01
Access Registrations List
  • Go to NKB Events → Events and select the event
  • Open the Registrations section for that event
  • You will see all students who have registered with their current status
02
Approve a Registration APPROVE
  • Click on a registration entry to open it
  • Review the student's form responses
  • Change the Status field to Approved
  • Click Save
✓ Once approved, the student can see the event as active and will receive any configured event pass.
03
Reject a Registration REJECT
  • Click on the registration entry you wish to reject
  • Change the Status field to Rejected
  • Optionally add a reason in the Notes field
  • Click Save
✕ Rejection can be reversed — set the status back to Approved at any time to reinstate the student.
📝

Prerequisite: The certificate template must be configured in the Poster Generation service first. A Certificate Name Enum will then be available — keep it handy.
01
Enable Certificate on the Event EVENT SETUP
  • While editing the event, locate the Is Certificate Enabled checkbox
  • Enable it to activate certificate generation — this is the master switch
  • If not enabled, no certificate will be generated regardless of other settings
02
Set Completion Criteria EVENT SETUP
FieldDescription
Resource IDContent unit ID tied to the event. Leave blank for virtual meetups.
Minimum Threshold (%)% of event completion required. E.g. 50% on a 2-hr event = attend 1 hr. Defaults to 1% if empty.
📌Students who don't meet the threshold will not receive a certificate.
03
Configure Certificate Template Data ADMIN PANEL
  • Copy the Event ID from the event you created
  • Go to Certificate Template Data in the Admin Panel
  • Fill in: Certificate Resource ID (paste Event ID), Template Data (JSON below), Enrollment Plans (NIAT)
{
  "certificate_template_type": "DEFAULT",
  "certificate_name_enum": "NIAT_MURFAI_CERTIFICATE"
}
📌The Certificate Name Enum must exactly match what was configured in Poster Generation. Any mismatch = no certificate generated.
