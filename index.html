Overview

This document covers the full admin workflow for running an event on the NxtWave platform. It is written for admin and ops staff who configure events from the Django Admin Panel. Follow the steps in order — later steps depend on the Event ID and form configuration completed in earlier ones.

The four core actions are: creating the event record, attaching a registration form, attaching a project submission form (if applicable), and managing participant registrations by approving or rejecting them.

📌
Prerequisite — Certificate template must exist first. If the event awards a certificate, the template must already be configured in the Poster Generation service via the Admin Panel before you begin event setup. Once done, note the Certificate Name Enum — you will need it when filling in the event form. If this has not been done, the certificate checkbox on the event will have no effect.
Access Required
Admin Panel login with NKB Events permissions
Time to Complete
~10–15 minutes for full event setup with both forms
Sections in Admin Panel
NKB Events → Events, Event Registrations
Key Output
Event ID (UUID from URL) — needed for certificate template data
Configuration Steps
01

Create the Event

The event record is the foundation for everything else. Complete and save this step first — the Event ID it generates is required for certificate configuration and form linking.

Admin Panel › NKB Events › Events › Add Event
Navigation
  1. 01 Log in to the Admin Panel with your admin credentials.
  2. 02 In the left sidebar, locate and click NKB Events to expand the section.
  3. 03 Click Events from the expanded list. The Events list page opens.
  4. 04 Click the Add Event button in the top-right corner of the page. The event creation form opens.
Filling in the Event Form
  1. 05 Event Name — Type the full name of the event exactly as students should see it on the platform.
  2. 06 Description — Click inside the description text area and enter the full event description. Include speaker details, agenda, format, and any other relevant information.
  3. 07 Start Date — Click the date field. A date picker appears. Select the date the event goes live. The field auto-fills on selection. You can also type the date directly in the format shown (e.g. 2026-06-24). Click Today if the event starts today.
  4. 08 Start Time — Click the time field and enter the start time. Click Now to use the current time.
  5. 09 Duration — Enter the length of the event in hours. For a 2-hour event enter 2. For 90 minutes enter 1.5.
  6. 10 Visibility Status — Click the dropdown and select PUBLIC so students can find and access the event. Select PRIVATE if you are still building it and want to publish later.
  7. 11 Is Carousel Enabled — Tick this checkbox if the event should appear in the rotating banner on the student homepage. When ticked, a Carousel Start Datetime field appears — set the date and time from which the carousel should start showing this event.
  8. 12 Is Certificate Enabled — Tick this checkbox if students earn a certificate upon completing the event. Only tick this if the certificate template is already configured in the Poster Generation service.
  9. 13 Participation Type — Click the dropdown. Select INDIVIDUAL if each student registers alone, or TEAM if students register as a group.
  10. 14 Short Text — Enter a one-line tagline or brief description. This appears on the event card in the student app.
  11. 15 Enroll Plans Str — Enter the enrollment plan identifier that can access this event. For NIAT events enter NIAT. Leave blank to allow all plans. If entering multiple plans, confirm the format with your tech lead before saving.
Saving & Getting the Event ID
  1. 16 Scroll to the bottom of the form. Click Save and continue editing — use this button, not plain Save, so you stay on the page.
  2. 17 The page reloads with a green success message at the top confirming the event was saved.
  3. 18 Look at the browser's address bar. The URL will look like:
    /admin/nkb_event/event/52/change/
    The number in the URL (e.g. 52) is your Event ID. Copy and save it — you will need it for certificate template configuration.

02

Add the Registration Form

The registration form is the sign-up form participants fill in before the event. It is configured inline on the same event detail page — no separate admin object is needed. Stay on the event detail page from Step 1 to continue.

Admin Panel › NKB Events › Events › [Your Event] › Registration Form Fields (inline)
Adding Fields
  1. 01 On the saved event detail page, scroll down past all the main event fields to the bottom of the page.
  2. 02 Find the inline section titled Registration Form Fields.
  3. 03 Click Add another Registration Form Field. A new row of inputs appears.
  4. 04 In the Field Label input, type what the participant will see — e.g. Full Name.
  5. 05 In the Field Type dropdown, select the appropriate input type. See the field type reference table below.
  6. 06 Tick the Is Required checkbox if this field must be completed before the form can be submitted.
  7. 07 If you selected Dropdown as the field type, an Options input appears. Enter each option on a separate line — e.g. one city per line.
  8. 08 Click Add another Registration Form Field again to add the next field. Repeat steps 04–07 for each field you need.
Setting the Registration Window
  1. 09 Find the Registration Open Date field. Set the date and time from which students can start submitting the registration form.
  2. 10 Find the Registration Close Date field. Set the date and time after which the form stops accepting new submissions. The form becomes inaccessible to students after this point.
  3. 11 Scroll to the bottom and click Save. The registration form is now live and attached to your event.

Recommended fields for most events: Full Name (Text, Required) · Email Address (Text, Required) · Mobile Number (Text, Required) · College / Organisation (Text, Optional). Only collect what you will actually use — shorter forms get higher completion rates.

03

Add the Project Submission Form

Configure this form only if your event requires participants to submit a project — for example, a hackathon or a coding challenge. For virtual meetups, podcasts, and events with no submission component, skip this step entirely.

Admin Panel › NKB Events › Events › [Your Event] › Project Submission Form Fields (inline)
  1. 01 Stay on the event detail page. Scroll below the Registration Form Fields section to find Project Submission Form Fields.
  2. 02 Click Add another Project Submission Form Field. A new row of inputs appears.
  3. 03 Set the Field Label, Field Type, and Is Required for each field — same process as the Registration Form in Step 2.
  4. 04 Add all the project fields your event requires. Refer to the recommended fields table below.
  5. 05 Set the Submission Deadline — the date and time by which all project submissions must be received. Submissions after this timestamp are not accepted. Set this carefully and confirm the deadline matches what was communicated to participants.
  6. 06 Scroll to the bottom and click Save.

04

Approve a Registration

Once participants begin submitting registrations, you review and approve them from the Event Registrations section. Approving a registration grants the student access to the event and triggers an automatic approval notification.

Admin Panel › NKB Events › Event Registrations › [Registration Entry]
  1. 01 In the Admin Panel left sidebar, under NKB Events, click Event Registrations.
  2. 02 The list shows registrations across all events. On the right side of the page, find the By Event filter panel. Click the name of your event to filter the list to only that event's registrations. The page reloads with filtered results.
  3. 03 You will see each registration row with the participant's name or email, the date submitted, and the current Status (Pending for new submissions).
  4. 04 Click on the row of the registration you want to review. The registration detail page opens.
  5. 05 Read through all the submitted form data — name, email, college, and any other fields you configured. Confirm the registration is valid.
  6. 06 Find the Status field. Click the dropdown and change the value from PENDING to APPROVED.
  7. 07 Leave the Rejection Reason field completely blank.
  8. 08 Scroll to the bottom and click Save.
  9. 09 The registration status updates to Approved. The student receives an automatic notification confirming their registration is approved and they now have access to the event.
✅
After saving, the status badge in the registrations list turns green and shows Approved. The student can now join the event from the platform.

05

Reject a Registration

Use the same navigation path as approval. The only difference is the status you set and the Rejection Reason you add. Always fill in the rejection reason — it keeps the record auditable.

Admin Panel › NKB Events › Event Registrations › [Registration Entry]
  1. 01 Navigate to NKB Events → Event Registrations. Apply the event filter as in Step 4.
  2. 02 Click on the registration entry you want to reject.
  3. 03 Review the submitted details carefully before proceeding.
  4. 04 Find the Status field. Click the dropdown and change the value to REJECTED.
  5. 05 Find the Rejection Reason field. Type a brief, clear reason — for example:
    Does not meet eligibility criteria for this event.
    This is shown internally, not to the student. Keep it factual and specific.
  6. 06 Scroll to the bottom and click Save.
  7. 07 The student receives an automatic rejection notification. The registration record remains in the list with a Rejected status badge for audit purposes.
When Approving
  • Status → APPROVED
  • Rejection Reason → leave blank
  • Student gets access notification
  • List badge turns green
When Rejecting
  • Status → REJECTED
  • Rejection Reason → fill in always
  • Student gets rejection notification
  • Record stays for audit trail
⚠️
Think before you save. The student receives a notification the moment you save — there is no confirmation step. If you accidentally reject a valid registration, you can reopen the record and change the status back to APPROVED, but the student will have already received the rejection message. Always double-check the correct registration is open before changing the status.

Student Experience

Here is what the student sees at each stage of the event lifecycle, from the moment the event is published to after it ends.

  • Before
    Event Published (Visibility = PUBLIC)
    The student can see the event listed on the platform. The event card shows the event name, start date, duration, and short text. If the carousel is enabled, the event also appears in the homepage banner.
  • Signup
    Registration Open
    During the registration window, a registration button appears on the event page. The student clicks it, fills in the registration form you configured, and submits. Their registration status is set to Pending until you act on it.
  • Approved
    After Approval
    The student receives a notification confirming their registration is approved. They can now see the Join Now button on the event page. If an Event Pass is configured, they also see a View Pass button alongside the event.
  • Rejected
    After Rejection
    The student receives a rejection notification. The event page reflects the rejected status. The student does not gain access to join the event.
  • Live
    During the Event
    The event status shows as Live Now. Approved participants see the Join Now button and can enter. The event duration counter is visible.
  • Ended
    After the Event Ends
    The status updates to Completed. If the student met the completion threshold and a certificate is configured, they see the message: "You've earned it! Your certificate will be available here soon." The certificate appears on the same screen by end of day. Students can share it directly to LinkedIn and Twitter.
Troubleshooting & Common Mistakes
The event is saved but students cannot see it on the platform.
Check the Visibility Status field on the event. It must be set to PUBLIC. If it is PRIVATE, open the event in the Admin Panel, change the status to PUBLIC, and save.
The registration form is not showing up for students.
Confirm that the Registration Open Date has passed and the Registration Close Date has not yet passed. The form is only visible within this window. Also verify that at least one form field has been added and saved.
The certificate is not being generated for students who attended.
Three things to check: (1) Is Certificate Enabled is ticked on the event. (2) The Certificate Template Data is configured in the Admin Panel with the correct Certificate Name Enum — it must exactly match what was set in the Poster Generation service. (3) The student met the Minimum Threshold configured in the Event Certificate Completion Criteria. See the separate Certificate KT Doc for full setup details.
I approved or rejected the wrong registration by mistake.
Reopen the registration record in the Admin Panel and change the Status to the correct value, then save. The status will update in the system. However, the student has already received the original notification — inform them directly if the first notification was incorrect.
I cannot find the Event ID after saving.
Open the event in the Admin Panel (NKB Events → Events → click your event). Look at the browser URL — the number between /event/ and /change/ is your Event ID. For example: /admin/nkb_event/event/52/change/ → Event ID is 52.
The project submission form fields are not saving.
Ensure every field row has a Field Label and a Field Type selected before saving. Rows with empty labels are silently skipped or may cause a validation error. Also confirm you clicked Save after adding the fields — inline sections are not auto-saved.
The event is not appearing in the homepage carousel.
Check two things: (1) Is Carousel Enabled is ticked. (2) The Carousel Start Datetime is set and the date/time has already passed. The carousel only picks up events after the start datetime.
