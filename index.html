01
Create an Event REQUIRED FIRST
  • Navigate to Django Admin → NKB Events → Events
  • Click "Add Event" to open the event creation form
  • Fill in all required fields: Event Name, Description, Start/End Date & Time, Visibility Status
  • Set Participation Type (Individual or Team)
  • Add Short Text for student-facing display
  • Set Enroll Plans Str to the appropriate enrollment plan (e.g. NIAT)
  • Click Save — the event is now created
📌 Copy and save the Event ID after saving — you will need it for certificate configuration later.
02
Add a Registration Form
  • While editing the event, locate the Event Registration Form section
  • Click "Add another Event Registration Form"
  • Select or create the form that students will fill when registering
  • Fields in the form typically include: Name, College, Year, Contact details
  • Save the event after adding the form
ℹ If no registration form is added, students can still register but will not be prompted to fill any details.
03
Add a Project Submission Form
  • In the event edit page, locate the Event Project Submission Form section
  • Click "Add another Event Project Submission Form"
  • Select the form students will use to submit their project work
  • Configure any deadline or submission window if required
  • Save the event
ℹ This step is optional — only add it if the event requires students to submit a project.
04
Approve a Registration
  • Go to NKB Events → Events → [Select Event] → Registrations
  • You will see a list of all students who have registered
  • Click on a registration entry to open it
  • Review the student's form responses
  • Change the Status to Approved
  • Click Save — the student's registration is confirmed
✓ Once approved, the student can see the event as active and will receive any configured event pass.
05
Reject a Registration
  • Go to NKB Events → Events → [Select Event] → Registrations
  • Click on the registration you wish to reject
  • Change the Status to Rejected
  • Optionally add a reason in the notes field
  • Click Save
✕ Rejected students will not be able to access the event. This action can be reversed by setting status back to Approved.

Module 02
Configuring Event Certificates
End-to-end setup for automatic personalized certificate generation upon event completion
▶ Module 02 — Certificate Configuration Walkthrough Watch before taking the quiz
Feature
Auto Certificate
Trigger
Event Completion
Config Steps
3 Steps
Delivery
By End of Day

Prerequisite

📌
Before configuring any event for certificate generation:
  • The certificate template must be configured in the Poster Generation service via the Admin Panel
  • Once configured, a Certificate Name Enum will be available — keep this handy
  • If the template is not set up in Poster Generation first, certificate configuration will not work
01
Enable Certificate on the Event EVENT SETUP

When setting up or editing an event, locate the checkbox labelled Is Certificate Enabled.

  • Enable this checkbox to activate certificate generation for the event
  • If not enabled, no certificate will be generated regardless of any other configuration
✕ This checkbox is the master switch — if it's off, Steps 2 and 3 have no effect.
02
Set Event Certificate Completion Criteria EVENT SETUP

This section defines what a student must meet to be eligible for the certificate. Configure two fields:

FieldDescriptionRequired
Resource ID The ID of the content unit tied to the event (e.g. a podcast unit ID). If it's a virtual meetup, leave blank. Conditional
Minimum Threshold (%) The % of completion a student must reach. E.g. 50% on a 1-hour event = must attend 30 min. Defaults to 1% if left empty. Required
📌 If the student does not meet the minimum threshold, the certificate will not be generated for them.
03
Configure Certificate Template Data ADMIN PANEL

This step links the certificate template to the event via the Certificate Template Data section in the Admin Panel.

  • Copy the Event ID from the event you created
  • Go to Certificate Template Data in the Admin Panel
  • Fill in the following three fields:
FieldWhat to Enter
Certificate Resource IDPaste the Event ID copied from Step 1
Template DataJSON with certificate_template_type and certificate_name_enum (see below)
Enrollment PlansEnter the relevant plan. For NIAT events, set to NIAT

Template Data JSON format:

{
  "certificate_template_type": "DEFAULT",
  "certificate_name_enum": "NIAT_MURFAI_CERTIFICATE"
}
📌 Always ensure the Certificate Name Enum exactly matches what was configured in the Poster Generation service. Any mismatch will result in no certificate being generated.

Student Experience

During the Event
  • Student can see the event is live and join it
  • If an Event Pass is configured, they also see their pass
After the Event
  • Eligible students see: "You've earned it! Your certificate will be available here soon."
  • Certificate appears on the same screen by end of day
  • Students can share directly to LinkedIn and Twitter

Module 03
Event Pass Generation
Configuring personalized event passes that students receive automatically upon registration
▶ Module 03 — Event Pass Configuration Walkthrough Watch before taking the quiz
Feature
Auto Event Pass
Trigger
Registration
Config Steps
3 Steps
Section
Event Pass Config

Prerequisite

📌
Before configuring the event pass:
  • The pass template must be configured in the Poster Generation service via the Admin Panel
  • Once configured, a Poster Name Enum will be available — keep this handy
  • If the pass template is not set up in Poster Generation first, the event pass configuration will not work
01
Enter the Poster Name Enum EVENT PASS CONFIG

Locate the Event Pass Config section while setting up or editing an event.

  • In the Poster Name Enum field, enter the exact enum value of the pass template configured in the Poster Generation service
  • This is how the system knows which pass template to use for this event
✕ The value must be exact — any typo or mismatch will cause the pass to not generate.
02
Select Overlay Options EVENT PASS CONFIG

Choose which student data gets overlaid onto the pass. The following checkboxes are available:

Overlay OptionWhat It Does
Overlay NameOverlays the student's name on the pass (field: name)
Overlay Profile PicOverlays the student's profile picture (field: profile_pic_url)
Overlay Event Start DateOverlays the event start date (field: event_start_date)
Overlay Event End DateOverlays the event end date (field: event_end_date)
Overlay Issued DateOverlays the date the pass was issued (field: issued_date)
📌 The checkboxes selected here must exactly match what was configured in the pass template in the Poster Generation service. Any mismatch will result in the pass not rendering correctly.
03
Set Fallback Avatar URL CONDITIONAL

The Fallback Avatar URL field applies when Overlay Profile Pic is enabled.

  • If a student has not uploaded a profile photo, the image from this URL appears on their pass instead
  • This field is only relevant when Overlay Profile Pic is selected
  • Make sure a valid default avatar image URL is provided
ℹ If Overlay Profile Pic is not selected, this field can be left blank.

Student Experience

✓
Once the event pass is correctly configured and the event goes live:
  • Students can see the event is live and join it
  • Alongside the event, students will also see their event pass with a "View Pass" button

Quick Reference Summary

FieldWhat to EnterNotes
Poster Name EnumEnum value from Poster Generation serviceMust be exact
Overlay OptionsTick fields that match the poster templateMismatch = pass won't render
Fallback Avatar URLURL of a default avatar imageOnly needed if Overlay Profile Pic is enabled
